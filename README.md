# טובתי — מעקב פוריות, בשבילך — גרסה 1.5.0

אפליקציית Android מקומית לרישום ידני של חום השחר (BBT), תצפיות יומיות ומחזור, עם גרף ומנוע ניתוח שמרני ושקוף. הממשק בנוי בעברית וב־RTL, עם משאבי אנגלית כחלופה, וללא חשבון משתמש, ענן, פרסומות או חיבור למדחום.

> האפליקציה אינה אמצעי מניעה, אינה מאשרת ביוץ בוודאות, אינה מוצר רפואי ואינה מחליפה ייעוץ רפואי. היא לעולם אינה מסמנת יום כ״בטוח״ או ״לא פורה״.

## מה כלול

- קליטה ידנית של טמפרטורה, שעה, אתר מדידה, שינה, הפרעות והערות.
- רישום דימום, מראה ותחושת נוזל צוואר הרחם, תצפית שקשה לפרש, בדיקת LH (כולל גבולי, שעה, מותג ורגישות אופציונליים) וכאב משוער.
- מעקב יומי אחר מצב רוח בברירה מרובה ובהערת מלל, מידת חשק מיני, אוננות, מגע מיני, מי יזמה, תסמינים גופניים וכדורים להקלה. אלו נשמרים להקשר ולזיהוי דפוסים אישיים, אך אינם מקבלים לבדם משקל בחיזוי הביוץ.
- תובנות אישיות עדינות במסך הבית על דפוסי מצב רוח, חשק וקרבה, תסמינים חוזרים, שימוש בכדורים ורצף תיעוד. נתונים חסרים אינם נחשבים לאפס והתובנות אינן אבחנה או המלצה רפואית/זוגית.
- התחלה ידנית של מחזור, מסך ״היום״, היסטוריה ועריכת רשומות.
- יומן מחזור מלא שמאורגן לפי ימי מחזור ולא לפי שבועות לועזיים: כל יום מציג מידע שנרשם בפועל, טווחים חזויים וסימני פוריות; לחיצה פותחת את כל מדידות החום, התצפיות, מצב הרוח, החשק, המגע, התסמינים וההערות של אותו יום.
- גרף BBT אינטראקטיבי עם הבחנה בין מדידות שנכללו והוחרגו.
- תחזית אישית רב־סימנית המשלבת היסטוריית מחזורים עם LH ונוזל צוואר הרחם שנרשמו במחזור הנוכחי.
- הפרדה בין שני ימי הביוץ העתידיים הסבירים ביותר, חלון הזדמנות להתעברות, ויום ביוץ יחיד שנבחר בדיעבד משינוי תרמי ו/או מ־LH שעבר או משיא נוזל שהושלם. מלוא משקלי אי־הוודאות נשמרים לחישוב ולהסבר, אך התצוגה נשארת ממוקדת.
- תחזית וסת: שני ימי התחלה סבירים לווסת הבאה, מעוגנים בהערכת הביוץ של המחזור הנוכחי ובשלב הלוטאלי האישי הנלמד מההיסטוריה, עם fallback להיסטוריית אורכי מחזור ובסיס הערכה מוצהר. הטווח מוצג במסך ״היום״, מסומן על הגרף ומלווה באורך השלב הלוטאלי המשוער לכל מחזור בהיסטוריה.
- בדיקת עבר (walk-forward backtest) במסך ההיסטוריה: כל מחזור שהושלם ושיש אחריו יום 1 שמור ורציף משוחזר יום אחר יום עם הנתונים שהיו ידועים באותו יום, ומדווחים שיעור פגיעת תחזית הווסת, המרחק החציוני וההתרעה המוקדמת החציונית. פער שנוצר ממחיקת מחזור אינו מומצא כתוצאת אמת.
- רמת מהימנות מילולית, הסבר לסימנים ששוקללו והצעת פעולה הבאה; המשקלים הם יחסיים ואינם אחוזי הסתברות רפואיים.
- זיהוי שינוי תרמי עמיד יותר לרעש: אתר מדידה קבוע למחזור, החרגה אוטומטית מהניתוח התרמי של מדידות לא אמינות וסריקת מועמדים חלופיים אם דפוס מוקדם הופרך.
- תזכורת בוקר מקומית ואופציונלית, המתוזמנת מחדש בכל יום לפי אזור הזמן ו־DST הנוכחיים.
- ייצוא CSV, גיבוי JSON מוצפן ושחזור מלא בשליטת המשתמשת. לאחר יצירת גיבוי אפשר לשלוח אותו כקובץ דרך חלון השיתוף/המייל; לפני שליחת גיבוי שאינו מהיום מוצעת יצירת גיבוי עדכני.
- נעילה ביומטרית או בקוד המכשיר וחסימת צילומי מסך — שתיהן אופציונליות.

האפליקציה מיועדת ל־Android 8.0 ומעלה (`minSdk 26`), ונבנית מול Android 16 / API 36. מזהה החבילה ב־release הוא `com.yv.bbttracker` וב־debug הוא `com.yv.bbttracker.debug`.

## טכנולוגיה וארכיטקטורה

- Kotlin 2.4.10, Java 17 ו־Jetpack Compose Material 3.
- Navigation 3 לניווט מבוסס routes טיפוסיים.
- Room למסד הנתונים, DataStore להגדרות ו־WorkManager לשרשרת תזכורות חד־פעמיות.
- Vico לגרף ו־Kotlin Serialization לגיבוי.
- ViewModel ו־StateFlow בזרימת נתונים חד־כיוונית.
- `AppContainer` להזרקת תלויות ידנית, ללא framework חיצוני.

פירוט נמצא ב־[ארכיטקטורה](docs/ARCHITECTURE_HE.md), ב־[אלגוריתם הממומש](docs/ALGORITHM_HE.md) וב־[דו״ח גרסה 1.5.0](docs/RELEASE_REPORT_1.5.0_HE.md). [מסמך יעד התחזית](docs/FORECAST_MODEL_HE.md) מרכז עקרונות רפואיים־הנדסיים וכיווני המשך; הוא אינו טענה שכל היעדים שבו כבר מומשו או שהמנוע עבר ולידציה קלינית.

## דרישות פיתוח

- JDK 17.
- Android SDK Platform 36 ו־Build Tools 36.
- Android Platform Tools (`adb`) להתקנה על מכשיר.
- אין צורך בהתקנת Gradle גלובלית; ה־wrapper מוריד Gradle 9.4.1.

ודאו ש־`JAVA_HOME` מצביע ל־JDK 17. צרו בשורש הפרויקט קובץ `local.properties` שאינו נכנס ל־Git:

```properties
sdk.dir=/absolute/path/to/Android/Sdk
```

## בנייה ובדיקות

משורש הפרויקט:

```bash
./gradlew --no-daemon clean
./gradlew --no-daemon testDebugUnitTest
./gradlew --no-daemon lintDebug
./gradlew --no-daemon assembleDebug
```

בדיקות היחידה מכסות בין היתר את מנוע ההיתוך, אירועי LH, נוזל צוואר הרחם, שינוי תרמי, איכות נתונים, מנוע התובנות, אימות טמפרטורה, תאימות גיבוי וקידוד CSV. בדיקות Android מכסות גם את migrations של Room מגרסה 1 ועד גרסה 4 ואת קובץ הגיבוי המוגן לשיתוף. ה־APK נוצר ב־`app/build/outputs/apk/debug/app-debug.apk` לאחר build מוצלח.

אפשר להריץ בפקודה אחת את שערי היחידה, lint ובניית debug, שאינם דורשים מכשיר:

```bash
./gradlew --no-daemon clean testDebugUnitTest lintDebug assembleDebug
```

בדיקות Android, לרבות migration של Room, דורשות emulator או מכשיר פעיל ורצות בנפרד:

```bash
./gradlew --no-daemon connectedDebugAndroidTest
```

## התקנה על מכשיר

הפעילו USB debugging במכשיר, אשרו את טביעת המחשב ובדקו שהמכשיר מזוהה:

```bash
adb devices
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n com.yv.bbttracker.debug/com.yv.bbttracker.MainActivity
```

לחלופין:

```bash
./gradlew installDebug
```

מומלץ לבדוק על מכשיר היעד גם עברית ו־RTL, מצב טיסה, מצב כהה, גופן מוגדל, תזכורת לאחר אתחול/ניהול סוללה, ייצוא ושחזור דרך אפליקציית Files ונעילה ביומטרית.

## חתימת release

צרו keystore ושמרו אותו במקום מאובטח שאינו בתיקיית Git. העתיקו את `keystore.properties.example` ל־`keystore.properties` והחליפו את הערכים. `storeFile` יכול להיות נתיב מוחלט, או נתיב יחסי לתיקיית `app`:

```bash
cp keystore.properties.example keystore.properties
./gradlew --no-daemon clean testDebugUnitTest lintRelease assembleRelease
```

הקובץ `keystore.properties` וקובצי keystore מוחרגים מ־Git. בלי אותו keystore לא ניתן לפרסם עדכון חתום לאותה התקנה. לפני הפצה יש להשלים את [רשימת בדיקות ה־release](docs/RELEASE_CHECKLIST_HE.md).

## פרטיות ונתונים

האפליקציה אינה מבקשת הרשאת אינטרנט, Bluetooth, מיקום או אחסון רחב. הנתונים נשמרים ב־sandbox של האפליקציה; גישה לקבצים לייצוא ולשחזור נעשית דרך בורר הקבצים של Android. ב־Manifest המקורי של האפליקציה מוצהרות במפורש רק הרשאות להתראות ולביומטריה. המניפסט הממוזג של APK כולל גם הרשאות תשתית שמוסיפות ספריות AndroidX עבור WorkManager ותאימות ביומטרית; הן מפורטות במסמך הפרטיות ואינן כוללות `INTERNET`. הרשאת ההתראות מתבקשת בזמן ריצה רק לצורך תזכורת, ואימות ביומטרי/קוד מכשיר מופעל רק אם נבחרה נעילה.

חשוב להכיר את הגבולות:

- מסד Room ו־DataStore אינם מוצפנים ברמת האפליקציה; הם נשענים על sandbox והצפנת המכשיר.
- גיבוי JSON בסכמת v4 מוצפן ב־AES-256-GCM עם מפתח הנגזר מסיסמה; השחזור תומך ב־schemas v1–v4 ומשלים עבור סכמות ישנות שדות חדשים בברירות מחדל שמרניות.
- CSV אינו מוצפן ומכיל מידע רגיש; יש לשמור ולשתף אותו בזהירות.
- אין שחזור סיסמה לגיבוי מוצפן.
- גיבוי המערכת של Android מנוטרל עבור האפליקציה.

הפירוט המלא נמצא ב־[מסמך הפרטיות והאבטחה](docs/PRIVACY_HE.md).
