# רשימת בדיקות לקראת Release 1.5.0

מסמך זה הוא checklist לביצוע על המקור ועל ה־APK המדויק שמותקנים במכשיר. סימון סעיף פירושו שנבדק בפועל; עצם קיום הקוד או הבדיקה אינו מספיק. את התוצאות הסופיות יש לרשום ב־[דו״ח גרסה 1.5.0](RELEASE_REPORT_1.5.0_HE.md).

## 1. גרסה וסביבת בנייה

- [ ] `versionName` הוא `1.5.0` ו־`versionCode` הוא `7`.
- [ ] `ENGINE_VERSION` הוא `bbt-fusion-2.2.0` בכל תוצאת ניתוח.
- [ ] `compileSdk` ו־`targetSdk` הם 36, `minSdk` הוא 26 ו־JDK 17 פעיל.
- [ ] Gradle wrapper מצביע ל־9.4.1 וכל גרסאות הספריות מגיעות מה־version catalog.
- [ ] אין תלויות alpha/beta/RC או repository לא מאושר.
- [ ] Room מוגדר כ־version 4, וארבעת ה־schemas המיוצאים v1–v4 נמצאים ב־`app/schemas`.
- [ ] `MIGRATION_1_2`,‏ `MIGRATION_2_3` ו־`MIGRATION_3_4` מחוברים ל־builder ואין `fallbackToDestructiveMigration`.
- [ ] סכמת גיבוי חדשה היא v4 וטווח השחזור הנתמך הוא v1–v4.

## 2. שערי איכות אוטומטיים

הרצה נקייה מומלצת:

```bash
./gradlew --no-daemon clean testDebugUnitTest lintDebug assembleDebug connectedDebugAndroidTest
```

- [ ] כל המשימות הסתיימו בהצלחה ללא התעלמות מכשל; מספרי הבדיקות נרשמו בדו״ח.
- [ ] בדיקות prior מכסות אפס היסטוריה, מחזור מלא אחד, כמה מחזורים, משקל עדכניות, פיזור לפי מהימנות ומחזור ארוך.
- [ ] בדיקות פלט מכסות הפרדה בין טווח ביוץ פרוספקטיבי, חלון התעברות וטווח רטרוספקטיבי, כולל חלון רחב משישה ימים בעת אי־ודאות.
- [ ] בדיקות LH מכסות חיובי ראשון, אות פעיל ביום החיובי ועד `+2` כולל, `+3` ללא חיובי חדש שאינו פעיל, חיובי נמשך ב־`+3/+4` שמרחיב את הטווח מהחיובי הראשון עד החיובי הנוכחי ועוד יום בלי לאפס את העוגן, מרווח של יותר משלושה ימים, שלילי מפריד ו־`BORDERLINE`.
- [ ] בדיקות נוזל צוואר הרחם מכסות מרקם, תחושה, בחירת האיכות הגבוהה, נוזל פורה היום, תרומה פרוספקטיבית של תצפית מהיום או מאתמול לפני שיא, `mucusObscured`, דפוס עולה ושיא שנקבע רק לאחר ירידה.
- [ ] בדיקות היתוך מכסות אות יחיד, כמה אותות מתיישבים, חיתוך טווחים, אותות סותרים, הרחבת טווח ומהימנות `LIMITED` בסתירה.
- [ ] בדיקות מהימנות מכסות `INSUFFICIENT`, `LIMITED`, `MODERATE` ו־`STRONG`, בלי לפרש את הקטגוריות כאחוזים.
- [ ] בדיקות שינוי תרמי מכסות חוסר baseline, גבולות `+0.10/+0.20`, fallback ביום הרביעי, יום חסר, יום רביעי נמוך ומועמד שאינו נשאר לאחר שחלף היום הרביעי.
- [ ] בדיקות תרמיות מכסות החרגת חום/מחלה, אתר שונה, פחות משלוש שעות שינה ומדידה שלא נעשתה מיד לאחר היקיצה.
- [ ] בדיקות תרמיות מכסות סריקת מועמד מאוחר אחרי הפרכת מוקדם, שתי טמפרטורות נמוכות לאחר אישור, דירוג התמדה וכמה מדידות ביום.
- [ ] בדיקות איכות מכסות ימים חסרים, הפרעות, שינוי אתר ופיזור שעות מעגלי סביב חצות.
- [ ] בדיקות `HistoricalCycleBuilder` מכסות מחזור עם BBT, LH, שיא נוזל, שילוב וסתירה.
- [ ] בדיקות backup v4 מכסות round-trip של כל השדות החדשים, סיסמה שגויה, tampering, schema לא נתמך ועברית.
- [ ] `BackupPayloadCompatibilityTest` מכסה payload v1, ברירות מחדל לשדות חדשים והשלמת `analysisSite`.
- [ ] בדיקות CSV מכסות יום עם תצפית בלבד, מרקם/תחושת נוזל, `mucusObscured`, תוצאת/שעת/מותג/רגישות LH, מצב רוח, חשק, מגע, תסמינים, פסיקים, מרכאות ושורות חדשות.
- [ ] בדיקות Room מכסות CRUD, uniqueness, בחירת מדידה בעסקה, חפיפת מחזורים ו־Flows.
- [ ] `AppDatabaseMigrationTest` משדרג schema v1 ל־v4 בלי אובדן נתונים, משלים אתר ידוע ובודק את ברירות המחדל של v2–v4.
- [ ] בדיקות התובנות מכסות ספי דגימה, ערכים חסרים, יוזמה, מצב רוח, חשק/מגע, תסמין חוזר, כדורים ורצף תיעוד.
- [ ] `ShareableBackupInstrumentedTest` מוודא שיתוף של מעטפת מוצפנת בלבד דרך `content://`.
- [ ] בדיקות UI מכסות onboarding, הוספה/עריכה/מחיקה, RTL, font scale, גרף, הרשאה שנדחתה ודיאלוגי גיבוי/מחיקה.
- [ ] בדיקות זמן מכסות DST בישראל, שינוי timezone, חצות ותזמון מחדש.

אם אין מכשיר או emulator זמין ל־instrumentation, אין לסמן את בדיקות Room migration/UI כהצלחה; יש לרשום אותן כממתינות.

## 3. בדיקת שדרוג v1.0.0 → v1.2.0

יש לבצע עם APK של 1.0.0 ו־1.2.0 החתומים באותו keystore:

- [ ] התקנת 1.0.0 נקייה והזנת לפחות שני מחזורים, מדידות מאתרים ידועים, תצפיות והגדרות.
- [ ] יצירת גיבוי מוצפן v1 לפני השדרוג ושמירתו מחוץ לאפליקציה.
- [ ] התקנת 1.2.0 באמצעות `adb install -r` ללא הסרת 1.0.0.
- [ ] האפליקציה עולה ללא crash וכל מחזור, מדידה, תצפית והגדרה קיימים.
- [ ] `analysisSite` הושלם לפי המדידה הנבחרת הראשונה בכל מחזור שבו קיימת מדידה כזו.
- [ ] תצפיות ישנות מקבלות תחושת נוזל ״לא נבדק״, `mucusObscured=false`, מטא־נתוני LH ריקים, מצב רוח/תסמינים ריקים וחשק/מגע במצב ״לא נרשם״ — ללא שינוי במשמעות הנתונים הישנים.
- [ ] הוספה ועריכה לאחר migration נשמרות לאחר process kill ו־relaunch.
- [ ] לאחר השדרוג ניתן ליצור גיבוי v3 ולשחזר אותו בהתקנה נקייה של 1.2.0.

## 4. בדיקת פונקציונליות רב־סימנית

- [ ] onboarding אינו מסתיים ללא קבלת הצהרת הבטיחות.
- [ ] הזנת `36.47` וגם `36,47` נשמרת ומוצגת נכון; ערכים מחוץ ל־`32.00–43.00` נחסמים.
- [ ] ניתן לערוך ולמחוק מדידה ולבחור מדידה יחידה לניתוח ביום עם כמה מדידות.
- [ ] אתר הניתוח ננעל למחזור ואינו משתנה בדיעבד לאחר שינוי ברירת המחדל.
- [ ] מדידת חום/מחלה, פחות משלוש שעות שינה או מדידה שסומנה שלא בוצעה מיד לאחר היקיצה נשמרת אך אינה מאשרת שינוי תרמי; סימון ״מדידה מאוחרת״ לבדו מוריד איכות אך אינו מוחרג אוטומטית.
- [ ] התחלת מחזור סוגרת את הקודם ללא חפיפה; תאריך לא חוקי מציג שגיאה.
- [ ] דימום, מרקם ותחושת נוזל, תצפית מוסתרת, LH גבולי/חיובי, שעה/מותג/רגישות, מצב רוח, הערת מצב רוח, חשק, מגע, תסמינים, כאב והערות נשמרים לאחר kill/relaunch.
- [ ] אוננות, מי יזמה מגע שסומן ״כן״, מספר כדורים והערת תרופה נשמרים ביומן, בהיסטוריה, בגיבוי וב־CSV.
- [ ] תובנה אינה מוצגת לפני סף הדגימה שלה ויום חסר אינו נחשב ״לא״ או אפס.
- [ ] מותג LH ריק/ארוך מדי, רגישות מחוץ לטווח ושעה לא תקינה נחסמים או מנורמלים לפי כללי הקלט.
- [ ] תוצאה גבולית מציעה בדיקה חוזרת; חיובי ראשון נשאר אות פעיל עד `+2`, וב־`+3` ללא חיובי חדש אינו פעיל. אם החיובי נמשך ב־`+3/+4`, העוגן אינו מתאפס והטווח מתרחב מהחיובי הראשון עד יום החיובי הנוכחי ועוד יום.
- [ ] נוזל פורה ממראה מימי/כמו חלבון ביצה או מתחושה רטובה/חלקלקה מעלה את רמת הפוריות ביום התצפית; תצפית שסומנה מוסתרת אינה מעלה אותה ואינה הופכת ליום שיא.
- [ ] תחזית רחבה קיימת גם ללא שלושה מחזורים; אחרי מחזור מלא אחד נראית התאמה אישית ראשונית ולא הבטחת דיוק.
- [ ] מסך היום מציג באופן מובחן טווח ביוץ אפשרי, חלון ניסיון להרות וטווח רטרוספקטיבי רק כאשר קיים.
- [ ] הגרף משתמש בשכבות נפרדות לחלון התעברות, תחזית פרוספקטיבית והערכה רטרוספקטיבית, בלי להציג אותן כאותו מושג.
- [ ] הגרף של המחזור הנוכחי נפתח סביב המדידה האחרונה גם ב־RTL וגם ב־LTR; הטווח העתידי נשאר נגיש בהחלקה אך אינו מסתיר את העקומה בפתיחה.
- [ ] ההיסטוריה מציגה הערכה רטרוספקטיבית ומהימנות למחזור מלא בלי לקרוא לה ״ביוץ מאושר״.
- [ ] מחזור שלאחריו פער ברשימת המחזורים מדולג מה־backtest ואינו מקבל תחילת וסת מדומה מ־`endDate + 1`.
- [ ] רמת המהימנות והפעולה הבאה מתעדכנות לאחר שינוי LH, נוזל או BBT.
- [ ] אותות סותרים מובילים להסבר זהיר, טווח רחב ומהימנות מוגבלת; אות אחד אינו דורס בשקט אות אחר.
- [ ] `BACKGROUND` אינו מוצג כ״לא פורה״ או כ״בטוח״.
- [ ] הממשק אינו מציג את המשקלים היחסיים כאחוז סיכוי ואינו טוען לדיוק, אישור רפואי או אבחנה.

- [ ] קובץ גיבוי v1 אמיתי מגרסה 1.0.0 נפתח ב־1.2.0, מוצג preview תקין ומשוחזר ללא אובדן נתונים ישנים.
- [ ] סיסמה שגויה, קובץ שנפגם, schema לא נתמך או מטא־נתוני LH לא חוקיים אינם משנים נתונים קיימים.

- [ ] CSV נפתח בתוכנה חיצונית ושומר עברית, שדות מצוטטים ושורות חדשות.
- [ ] עמודות CSV החדשות תואמות לנתונים באפליקציה; שדה חסר נשאר חסר ואינו מומצא כשלילי/יבש.

## 6. פרטיות ו־Manifest

- [ ] אין בקוד Firebase, analytics, ads, Crashlytics, endpoint חיצוני או logging של נתוני בריאות.
- [ ] `android:allowBackup="false"`, `android:usesCleartextTraffic="false"` ו־`android:supportsRtl="true"` קיימים.
- [ ] אין הרשאות `INTERNET`, `BLUETOOTH*`, `ACCESS_*_LOCATION`, `READ_*` או `WRITE_*`.
- [ ] הרשאות ה־APK תואמות ל־allowlist: התראות וביומטריה, ולפי המניפסט הממוזג גם הרשאות AndroidX/WorkManager המוסברות (`USE_FINGERPRINT`, ‏`WAKE_LOCK`, ‏`RECEIVE_BOOT_COMPLETED`, ‏`FOREGROUND_SERVICE`, ‏`ACCESS_NETWORK_STATE`); אין `INTERNET` או הרשאות בריאות/מיקום/אחסון בלתי צפויות.
- [ ] גיבוי נכתב רק לאחר הצפנה; סיסמה אינה נשמרת ואינה נכתבת ללוג.
- [ ] התראה אינה חושפת טמפרטורה, מחזור או מצב פוריות במסך הנעילה.
- [ ] מחיקת כל הנתונים דורשת אישור, מבטלת תזכורת ומאפסת Room ו־DataStore.

בדיקת ה־APK בפועל:

