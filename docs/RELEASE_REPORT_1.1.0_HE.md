## שינויי המוצר הממומשים

### תחזית וניתוח

- מנוע התחזית הוחלף מניתוח לוח שנה נפרד ל־fusion דטרמיניסטי של היסטוריה אישית, LH, ריר צוואר הרחם ו־BBT.
- prior רחב זמין גם בתחילת שימוש, והתאמה אישית מתחילה מהמחזור המלא הראשון. אין עוד חסימה קשיחה עד שלושה מחזורים.
- הפלט מפריד בין טווח ביוץ פרוספקטיבי, חלון הזדמנות להתעברות וטווח ביוץ רטרוספקטיבי. הטווח הרטרוספקטיבי יכול להיגזר משינוי תרמי ו/או מאירוע LH שעבר או משיא ריר שהושלם; אותות שמתיישבים כרונולוגית מצמצמים ומחזקים אותו.
- נוספו מהימנות מילולית, רמת פוריות שימושית לתכנון, רשימת אותות והצעת פעולה הבאה.
- משקלי התאריכים נשארים relative weights פנימיים ואינם מוצגים כהסתברויות רפואיות.
- סתירה בין BBT, LH וריר מרחיבה את הטווח ומגבילה את המהימנות במקום לתת לאות אחד לדרוס את האחר.

### קלט רב־סימני

- לריר נוספו תחושה (`NOT_CHECKED`, `DRY`, `DAMP`, `WET`, `SLIPPERY`) וסימון שהתצפית מוסתרת או אינה אמינה לניתוח.
- ל־LH נוספה תוצאה `BORDERLINE`, וכן שעה, מותג ורגישות אופציונליים.
- אירוע LH מעוגן בחיובי הראשון ונחשב פעיל ביום החיובי ועד `+2`, כולל. ב־`+3` ללא חיובי חדש הוא אינו עוד אות פרוספקטיבי פעיל. אם נרשם חיובי נמשך ב־`+3/+4`, העוגן אינו מתאפס; הטווח הפרוספקטיבי מתרחב מהחיובי הראשון עד יום החיובי הנוכחי ועוד יום כדי לבטא אי־ודאות ולא להתעלם מהאות העדכני.
- ריר מנתח מראה ותחושה יחד ולוקח את האיכות הגבוהה מביניהם. מראה מימי/שקוף־נמתח או תחושה רטובה/חלקלקה, כאשר התצפית אינה מוסתרת, נחשבים ריר פורה ביום התצפית. תצפית פורה מהיום או מאתמול יכולה להשפיע פרוספקטיבית לפני השלמת השיא; שיא נקבע רק לאחר ירידה מתועדת.

### עמידות BBT

- אתר הניתוח נשמר במחזור ואינו משתנה בדיעבד עם ברירת המחדל.
- מדידה שנבחרה אך כוללת חום/מחלה, אתר שונה, פחות משלוש שעות שינה או סימון שלא נמדדה מיד לאחר היקיצה אינה משמשת לאישור שינוי תרמי.
- הגלאי סורק את כל המועמדים, דוחה מועמד fallback אם היום הרביעי חלף והוא חסר/נמוך, ובוחן התמדה בארבעת הימים שלאחר אישור.
- שתי טמפרטורות נמוכות בתקופת ההתמדה מפריכות דפוס מוקדם ומאפשרות לשקול מועמד מאוחר יותר.
- פיזור שעת המדידה מחושב בצורה מעגלית סביב חצות.

### ממשק ותצוגה

- מסכי היום, הגרף וההיסטוריה מקבלים את שדות הפלט החדשים ואת תוצאות הניתוח ההיסטורי המאוחד.
- הגרף מציג בשכבות נפרדות, כאשר הטווחים קיימים, את חלון ההתעברות, התחזית הפרוספקטיבית וההערכה הרטרוספקטיבית.
- הניסוחים נשארים לא־קליניים: אין יום ״בטוח״/״לא פורה״, אחוזי סיכוי או ״ביוץ מאושר״.

## נתונים ותאימות

### Room v2

`MIGRATION_1_2` מוסיף:

- `cycles.analysisSite` וממלא אותו מהמדידה הנבחרת הראשונה במחזור כאשר קיימת;
- `daily_observations.mucusSensation` עם `NOT_CHECKED`;
- `daily_observations.mucusObscured` עם `false`;
- `lhTestMinutesOfDay`, `lhTestBrand` ו־`lhTestSensitivityMilliIu` כערכים nullable.

schemas v1 ו־v2 מיוצאים. המסד אינו משתמש ב־destructive migration.

### גיבוי ו־CSV

- גיבוי חדש נכתב כ־schema v2 עם `appVersion=1.1.0` ומכיל את השדות החדשים.
- השחזור מקבל payload v1 ו־v2. payload v1 מפוענח עם ברירות מחדל לשדות החדשים, ו־`analysisSite` מושלם לפי המדידה הנבחרת הראשונה במחזור כאשר אפשר.
- בדיקות העקביות כוללות שעה, אורך מותג וטווח רגישות LH.
- CSV כולל גם ימים עם תצפית ללא טמפרטורה, ואת עמודות מראה/תחושת/הסתרת ריר, תוצאת/שעת/מותג/רגישות LH וכאב ביוץ.
- `prediction_snapshots` נשאר נתון נגזר ואינו נכלל בגיבוי.

## תוצאות בדיקות סופיות

הבדיקות הורצו על אותו snapshot מקומי שממנו נבנה ה־APK. הן כוללות את מנוע התחזית, גבולות LH, רציפות ריר, BBT, גיבוי, Room migration ושינוי גבולות מחזור.

| שער | פקודה/מקור | תוצאה |
|---|---|---|
| Unit tests | `./gradlew --no-daemon :app:testDebugUnitTest` | עבר — `89/89`, אפס כשלונות ושגיאות |
| Debug lint | `./gradlew --no-daemon :app:lintDebug` | עבר |
| Debug APK | `./gradlew --no-daemon :app:assembleDebug` | עבר |
| Instrumentation / Room migration | `./gradlew --no-daemon :app:connectedDebugAndroidTest` | עבר — `11/11` על Android 16 / API 36 |
| Release lint | `./gradlew --no-daemon lintRelease` | עבר |
| Signed release APK | `./gradlew --no-daemon assembleRelease` | עבר עם R8 ו־resource shrinking |

ההרצה הנקייה המשולבת הסתיימה ב־`BUILD SUCCESSFUL`; לאחריה `scripts/build_release.sh` סיים בהצלחה את בדיקות היחידה, release lint, הבנייה, שער ההרשאות ואימות החתימה.

## מגבלות ושארית סיכון

- `bbt-fusion-2.0.0` לא עבר ולידציה פרוספקטיבית מול אולטרסאונד או ground truth רפואי. אין לגרסה 1.1.0 נתון sensitivity, specificity או שיעור כיסוי מאומת.
- הקטגוריות `LIMITED`/`MODERATE`/`STRONG` הן דירוג תמיכה פנימי, לא הסתברות ביוץ או התעברות.
- אין שמירת snapshot תחזית יומי בשימוש הרגיל ואין עדיין backtesting מובנה; לכן לא ניתן למדוד מהאפליקציה עצמה את התחזית כפי שנראתה בכל יום בעבר.
- נשמרת תוצאת LH אחת לכל תצפית יומית; אין סדרת בדיקות מרובות באותו יום או ניתוח יחס קווים כמותי.
- מותג ורגישות LH נשמרים אך אינם מכיילים כרגע את משקל האות.
- Room ו־DataStore אינם מוצפנים ברמת האפליקציה ונשענים על sandbox והצפנת המכשיר. CSV אינו מוצפן.
- BBT, LH וריר הם אותות ביתיים עקיפים ומושפעים מאיכות המדידה ומהקשר אישי. האפליקציה אינה מכשיר רפואי ואינה מיועדת למניעת הריון.

[מסמך יעד התחזית](FORECAST_MODEL_HE.md) מפרט עקרונות מחקריים והמשך אפשרי. [מסמך האלגוריתם](ALGORITHM_HE.md) הוא המקור להתנהגות הממומשת ב־1.1.0; במקרה של הבדל, אין לפרש את מסמך היעד כתכונה שכבר קיימת.
